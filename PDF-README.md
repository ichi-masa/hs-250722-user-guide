# PDF自動生成ガイド

このドキュメントでは、HTMLファイルからPDFを自動生成する方法を説明します。

## 概要

このプロジェクトでは、HTMLファイルから自動的にPDFを生成するNode.jsスクリプトが用意されています。
HTMLを翻訳するだけで、PDFも自動的に多言語対応されます。

**2つのモードがあります：**

| モード | コマンド | 出力 | リンク |
|---|---|---|---|
| 個別PDF | `node generate-pdf.js` | 1HTMLページ = 1PDFファイル（34ファイル） | △ ビューア依存 |
| 結合PDF | `node generate-pdf.js --combined` | 全ページを1つのPDFに結合 | ◎ PDF内ジャンプ |

## 個別PDFと結合PDFの比較

> **おすすめ**: ページ間のリンク操作を重視する場合は**結合PDF**がおすすめです。個別PDFではブラウザの環境によってリンクの挙動が異なります（後述）。結合PDFであれば同一ファイル内でジャンプするため、どの環境でもストレスなく閲覧できます。

### 個別PDF

1HTMLページ = 1PDFファイルとして出力します。

**メリット:**
- WEBサイトのPC表示をほぼそのまま再現（サイドナビ・レイアウト含む）
- ファイルごとに軽量で、必要なページだけ共有・閲覧できる
- `dist` フォルダの中身をそのままサーバーにアップロードすれば、HTMLと同じ構成でPDFを確認できる（`assets/pdf/` 内に全PDFが格納済み）

**デメリット:**
- ブラウザにAdobe Acrobat拡張機能がインストールされている場合、リンクが別タブで開き、毎回「Webページを開く」の確認ダイアログが表示されます（Chrome内蔵のPDFビューアでは同一タブで開き、ダイアログも表示されません）
- サーバー上でリンクを機能させるには `--base-url` の指定が必要

### 結合PDF

全ページを1つのPDFファイルにまとめます。

**メリット:**
- 1ファイルで全ページを閲覧でき、共有が簡単
- リンクがPDF内ジャンプになるため、同一タブ内でスムーズに遷移できる
- サーバー設置不要、ローカルでもリンクが機能する

**デメリット:**
- ファイルサイズが大きい（全ページ結合のため数MB〜）
- WEBサイトと完全に同じ見た目にはならない

### 結合PDFがWEBサイトと完全に同じにならない理由

結合PDFでは全ページを1つのHTMLに統合してからPDF化するため、以下の違いが生じます：

- **ページサイズの制約**: PDFは固定サイズのページに分割されるため、WEBのように可変長のスクロールレイアウトが再現できません。ページの区切り位置でコンテンツが分かれる場合があります
- **画像サイズの調整**: WEBではレスポンシブに表示される画像が、PDFのページ幅に合わせて固定サイズになります。元のWEB表示とサイズが異なる場合があります
- **CSSの互換性**: PDFレンダリングエンジン（Chromiumベース）はWEBブラウザと挙動が異なる部分があり、CSS Grid の subgrid やスクロール系の機能はPDFでは再現されません
- **インタラクティブ要素の非対応**: 検索機能、言語切り替え、スクロールヒントなどの動的要素はPDFでは動作しません

## 必要な環境

- **Node.js** バージョン18以上
  - [Node.js公式サイト](https://nodejs.org/)からダウンロードしてインストールしてください
  - インストール確認：`node -v` でバージョンが表示されればOK

## 必要なファイル

納品ファイルに以下が含まれています：

```
プロジェクトフォルダ/
├── generate-pdf.js              ← PDF生成スクリプト
├── package.json                 ← 依存パッケージ情報
├── package-lock.json            ← バージョン固定ファイル
└── dist/
    ├── en-GB/
    │   ├── index.html
    │   └── assets/pdf/          ← PDFの出力先
    ├── ja-JP/
    │   └── ...
    └── ... (他の言語)
```

## 使い方

### ステップ1：初回のみ（環境セットアップ）

プロジェクトフォルダで以下のコマンドを実行してください：

```bash
npm install
```

このコマンドで必要なツール（puppeteer等）がインストールされます。
初回のみ実行が必要です。

### ステップ2：HTMLを翻訳

`dist`フォルダ内のHTMLファイルを翻訳します。

### ステップ3：PDFを生成（毎回実行）

プロジェクトフォルダで以下のコマンドを実行します：

#### 個別PDF（1ページ = 1ファイル）

```bash
node generate-pdf.js
```

各HTMLページがそれぞれ1つのPDFファイルとして出力されます。
WEBサイトのPC表示と同じ見た目です。

#### 個別PDF + サーバー設置用リンク

PDFをWEBサーバーに設置する場合は、`--base-url` オプションで設置先のURLを指定してください。
PDF内のリンクが指定URLの絶対パスに変換され、サーバー上でリンクが正しく機能します。

```bash
node generate-pdf.js --base-url=https://example.com/pdf/
```

- URLの末尾に `/` を付けてください
- 指定しない場合、リンクはファイル名のみ（`filename.pdf`）になります
- **設置先のURLが変わった場合は、新しいURLを指定して再生成が必要です**

#### 結合PDF（全ページ1ファイル）

```bash
node generate-pdf.js --combined
```

全ページが1つのPDFにまとまります。
サイドナビ付きで、リンクをクリックするとPDF内でページジャンプします。

## 多言語対応

このスクリプトは自動的に全言語に対応します：
- `dist`フォルダ内の全ての言語フォルダを自動検出
- 各言語ごとにPDFを生成
- 新しい言語を追加した場合も自動で対応
- サイドナビのタイトルも翻訳済みHTMLから自動取得

## 生成されるファイル

### 個別PDFモード

各言語フォルダの `assets/pdf/` にHTMLと同名のPDFが生成されます：

```
dist/
├── en-GB/
│   └── assets/pdf/
│       ├── index.pdf
│       ├── important-safety-information.pdf
│       ├── how-to-use.pdf
│       ├── how-to-use-know-your-device.pdf
│       └── ...（34ファイル）
├── ja-JP/
│   └── assets/pdf/
│       ├── index.pdf
│       └── ...
└── ...
```

### 結合PDFモード

各言語フォルダに1つのPDFが生成されます：

```
dist/
├── en-GB/
│   └── assets/pdf/
│       └── user-guide-en-GB.pdf
├── ja-JP/
│   └── assets/pdf/
│       └── user-guide-ja-JP.pdf
└── ...
```

## PDFの特徴

- **テキスト選択可能**: PDFビューアでテキストを選択・コピーできます
- **見栄え維持**: WEBサイトのPC表示と同じ見た目で出力されます
- **画像・テーブル対応**: 画像やテーブルもそのまま表示されます
- **結合PDFのリンク**: サイドナビや本文中のリンクがPDF内でジャンプします

## PDF内リンクの挙動について

### 個別PDF

PDF内のリンク（他ページへの参照）は、リンク先のPDFファイルをブラウザで開きます。
ただし、使用しているPDFビューアによって挙動が異なります：

| 環境 | リンクの開き方 | 確認ダイアログ |
|---|---|---|
| Chrome内蔵PDFビューア | 同一タブで開く | 表示されない |
| Adobe Acrobat拡張機能 | 別タブで開く | 毎回表示される |

Adobe Acrobat拡張機能を無効にするか、シークレットモード（拡張機能が無効化される）で閲覧すると、Chrome内蔵ビューアで快適に閲覧できます。

### サーバー設置時の注意

個別PDFをWEBサーバーに設置してリンクを機能させるには：

1. **全てのPDFファイルを同じディレクトリに設置**してください
2. **`--base-url` に設置先のURLを指定**してPDFを生成してください
3. 設置先のURLが変わった場合は、再生成が必要です

```
サーバー上のディレクトリ構成（例）:
https://example.com/pdf/
├── index.pdf
├── important-safety-information.pdf
├── how-to-use.pdf
├── how-to-use-know-your-device.pdf
└── ...（全PDFファイル）
```

### 結合PDF

- 結合PDFでは全ページが1ファイル内にまとまっているため、リンクはPDF内アンカージャンプになります
- 同一タブ内でジャンプし、別タブでは開きません
- サーバー設置不要でローカルでもリンクが機能します

## トラブルシューティング

### エラー: `distディレクトリが見つかりません`

`dist`フォルダが存在することを確認してください。

### エラー: `puppeteer が見つかりません`

以下のコマンドを実行してください：

```bash
npm install
```

### エラー: `node コマンドが見つかりません`

Node.jsがインストールされていません。
[Node.js公式サイト](https://nodejs.org/)からダウンロードしてインストールしてください。

### PDFの生成に時間がかかる

全言語分を一括生成するため、言語数が多い場合は数分かかることがあります。
1言語あたり約30秒〜1分程度です。
